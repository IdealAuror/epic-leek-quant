## 一、命名规范

### spec_id

```
<Phase>-F<因子号>-<变体>-<季度>-v<版本>
```

| 段 | 取值 | 示例 |
|----|------|------|
| Phase | P1 / P2 / P3 / P4 | P1 |
| 因子号 | F1–F5（对应五大因子） / MF（多因子合成） / ST（压测） / LP（实盘） | F1 |
| 变体 | V1–V5（见 thinking-prompt.md §四）/ BASE（基线） | V1 |
| 季度 | YYYYQn | 2026Q2 |
| 版本 | v1, v2, ...（修订递增） | v1 |

完整示例：`P1-F1-EV-2026Q2-v1` = Phase 1 第一个因子 EV 的 2026Q2 第一版 spec。

### 文件命名

| 类型 | 路径 | 文件名 |
|------|------|--------|
| spec | `research/specs/` | `<spec_id>.md` |
| 偏差记录 | `research/reports/` | `<spec_id>-deviation.md` |
| 代码审核 | `research/reports/` | `<spec_id>-code-review.md` |
| 结果判定 | `research/reports/` | `<spec_id>-adjudication.md` |
| 执行代码 | `joinquant/strategies/` | `<spec_id>.py` |
| 回测结果 | `results/` | `<spec_id>/`（目录，含 CSV） |
| 决策记录 | `research/decisions/` | `<spec_id>-decision.md` |
