## 一、假设

```yaml
hypothesis: |
  Phase 2 验证的 F2+F5 Baseline（P1-F2F5-EP-LowVol-2026Q2-v1）：
  (1) 在 Purged Walk-Forward CV 下不过拟合（样本外 Sharpe 不显著下降）
  (2) 在制度断点（2019.06 注册制 / 2024.02 量化踩踏 / 2024.04 新国九条）后
      表现方向不反转
  (3) 在考虑 F1-F5 五因子多重比较修正后，F2 alpha 仍显著（DSR 通过）
  (4) 【新增】用 F5 信号做目标波动率仓位管理，能把回撤从 47% 降到 30% 以内，
      且年化收益下降不超过 3pp（回撤换收益的合理 trade-off）

economic_intuition: |
  Phase 3 增强版的核心创新：把 F5 LowVol 从"选股层"延伸到"仓位层"。
  
  Phase 2 发现 F5 是极有效的风险调节器：
    - F2 单因子 527% → F2+F5 组合 1237%（收益翻倍）
    - F2 单因子回撤 21% → F2+F5 组合回撤 18%（回撤降低）
  
  这意味着 F5 信号本身就携带了市场风险状态的信息。Phase 3 模块 4 用 F5 信号
  做仓位调节：
    - 当全市场 LowVol 信号弱（高波动股占比高 = 市场高波动）时减仓
    - 当全市场 LowVol 信号强（低波动股占比高 = 市场低波动）时加仓
    - 目标波动率 = 15%，实际波动率超过时按比例降仓位
  
  这与 Phase 1 的"PE 锚定择时"完全不同：
    - PE 锚定择时：对个股选股不对症（已证明失败）
    - 目标波动率仓位：组合层面风控，用 F5 信号作为市场状态代理

failure_condition: |
  明确失败条件：
  - Walk-Forward CV 样本外 Sharpe < 0.2（过拟合，样本内表现无法复制）
  - 任一制度断点后表现方向反转（如 2019.06 后年化收益转负）
  - DSR 修正后 F2 alpha 不显著（多重比较偏差主导，F2 是 data-mining 产物）
  - 目标波动率仓位管理后回撤仍 > 35%（仓位管理无效）
  - 目标波动率仓位管理后年化收益下降 > 5pp（仓位管理过度牺牲收益）
```

## 三、四个模块设计

```yaml
modules:
  module_1_purged_walk_forward:
    description: |
      Purged Walk-Forward Cross-Validation（防泄漏交叉验证）。
      用滚动窗口训练-验证，验证 F2+F5 Baseline 是否过拟合。
    method: |
      - 训练窗：60 个月（5 年）
      - 验证窗：12 个月（1 年，紧接训练窗）
      - Purging：验证窗前后各 3 个月剔除（避免信息泄漏，_fwd_return 重叠）
      - Embargoing：验证窗后 1 个月不参与下一轮训练
      - 滚动步长：12 个月
      - 每轮在训练窗优化参数（持仓数 30/50/100，F5 权重 0.3/0.5/0.7），
        在验证窗测试样本外表现
    outputs:
      - "样本内 vs 样本外 Sharpe 对照表"
      - "参数稳定性：每轮最优参数的变化"
      - "样本外累计净值曲线"
    pass_criteria:
      - "样本外 Sharpe ≥ 0.2"
      - "样本外 Sharpe / 样本内 Sharpe ≥ 0.5（不过拟合）"
      - "最优参数非边际极端值（如持仓数 ≠ 30 或 100 极端值）"

  module_2_regime_breakpoints:
    description: |
      按 A 股制度断点分段，验证 F2+F5 Baseline 跨断点稳健性。
    breakpoints:
      - {date: 2017-06, name: "MSCI 纳入 A 股"}
      - {date: 2019-06, name: "注册制科创板开板"}
      - {date: 2024-02, name: "量化 DMA 踩踏"}
      - {date: 2024-04, name: "新国九条"}
    method: |
      每个断点前后各取 2 年数据，分别跑 F2+F5 Baseline 回测，
      对比年化收益/Sharpe/回撤/Alpha 方向。
    outputs:
      - "各断点前后表现对照表"
      - "断点后表现是否方向反转"
    pass_criteria:
      - "任一断点后年化收益不转负"
      - "断点后 Sharpe 下降幅度 < 50%"
      - "Alpha 方向不反转"

  module_3_dsr:
    description: |
      Deflated Sharpe Ratio（多重比较修正）。
      考虑 F1-F5 五因子多重检验后，F2+F5 Baseline 的 Sharpe 是否仍显著。
    method: |
      DSR 公式（López de Prado 2014）：
        DSR = (SR_observed - E[max(SR)] under multiple testing) / SE(SR)
      其中：
        - SR_observed = Baseline 实测 Sharpe
        - E[max(SR)] = 多重检验下最大 Sharpe 的期望，依赖：
          - N = 试验次数（F1-F5 = 5 因子 + F2+F5 组合 = 6 次试验）
          - T = 样本长度（144 月）
          - skew/kurt of returns
        - SE(SR) = Sharpe 的标准误
      DSR ≥ 0.95 表示 Sharpe 在多重检验下仍显著
    outputs:
      - "DSR 值"
      - "多重检验前后 Sharpe 显著性对照"
    pass_criteria:
      - "DSR ≥ 0.95（多重比较修正后仍显著）"

  module_4_target_volatility:
    description: |
      【新增创新】目标波动率仓位管理。
      用 F5 信号作为市场状态代理，动态调节组合仓位。
    method: |
      仓位规则：
        (1) 计算全市场 LowVol 信号代理：
            market_vol_signal = 横截面 mean(F5_vol) 的 z-score
            （全市场低波动股占比高 = 市场低波动 = 加仓）
        (2) 计算组合实际波动率：
            realized_vol = 过去 60 交易日组合日收益年化波动率
        (3) 目标仓位：
            target_weight = min(target_vol / realized_vol, max_leverage)
            其中 target_vol = 15%, max_leverage = 1.0（不加杠杆）
        (4) F5 信号叠加：
            当 market_vol_signal < -1（市场高波动）时，
            target_weight *= 0.5（额外减仓 50%）
            当 market_vol_signal > 1（市场低波动）时，
            target_weight *= 1.0（不额外加仓，避免追高）
      仓位调节频率：月度（与调仓解耦，每月初调仓位）
    variants:
      - {id: V1, name: "纯目标波动率（无 F5 信号叠加）", target_vol: 0.15}
      - {id: V2, name: "F5 信号叠加目标波动率", target_vol: 0.15, f5_overlay: true}
      - {id: V3, name: "保守版（目标波动率 12%）", target_vol: 0.12, f5_overlay: true}
    outputs:
      - "V1/V2/V3 vs Baseline 收益风险对照表"
      - "仓位时序曲线（看仓位调节是否合理）"
      - "回撤降低幅度对照"
    pass_criteria:
      - "V2 回撤 < 35%（核心目标）"
      - "V2 年化收益下降 < 3pp（收益-回撤 trade-off 合理）"
      - "V2 Sharpe 不下降"
```

## 四、数据处理

```yaml
data_processing:
  stock_pool:
    - 全A（get_all_securities）
    - 剔除 ST/*ST
    - 剔除次新股（上市不满 180 天）
    - 剔除金融股
  filters:
    - "net_profit > 0"
    - "debt_to_assets <= 1.0"
  winsorize: 1%/99% 分位数裁剪
  zscore: 横截面 Z-score 标准化
  neutralize:
    - size  # log(market_cap) OLS 残差（与 Phase 2 一致）
  missing: drop_if_any_critical_missing
```
