## 一、假设

```yaml
hypothesis: |
  Phase 3 验证的 F2+F5 Baseline：
  (1) 扣除保守交易成本（万三佣金 + 印花税 + 千一滑点）后仍有正期望
      （年化净收益 > 5%，Sharpe > 0.3）
  (2) 5000 万资金下的冲击成本可承受（< 净收益的 20%）
  (3) F2/F5 因子拥挤度未进入历史 90 分位警戒
  (4) 换手-alpha 闭环合理（边际信息收益 ≥ 边际交易成本）
  (5) 【仓位管理改进】用最大回撤约束 + 动态止损，能把日频回撤从 47% 降到 30% 以内，
      且年化收益下降不超过 5pp

economic_intuition: |
  Phase 4 是从"理论有效"到"实战可用"的最后一公里。
  
  Phase 3 暴露的核心问题：
  1. 回撤 47.62% 实战不可接受（私募产品通常 <30%）
  2. 目标波动率方法降波动不降回撤（路径依赖）
  3. 策略近期表现恶化（2025-2026 样本外 Sharpe -0.316）
  
  Phase 4 的核心创新：仓位管理从"波动率控制"改为"回撤控制"。
  - 最大回撤约束：当组合回撤超过 15% 时减仓，回撤修复后加仓
  - 动态止损：组合从高点回撤 10% 时触发止损线，降至 50% 仓位
  - 这与 Phase 3 的目标波动率完全不同：直接控回撤，不控波动

failure_condition: |
  明确失败条件：
  - 成本后年化净收益 < 5%（成本吞噬 alpha）
  - 5000 万资金冲击成本 > 净收益 30%（容量不足）
  - F2/F5 因子拥挤度进入历史 90 分位（策略已拥挤）
  - 换手-alpha 闭环：边际交易成本 > 边际信息收益（换手过度）
  - 仓位管理后回撤仍 > 35%（回撤控制无效）
  - 仓位管理后年化下降 > 8pp（过度牺牲收益）
```

## 三、五个模块设计

```yaml
modules:
  module_1_net_return:
    description: |
      成本后净收益分析（毛 vs 净）。
      在 Phase 1 日频回测基础上，加入交易成本，对比毛收益与净收益。
    method: |
      - 复用 Phase 1 F2+F5 Baseline 日频回测
      - 每次调仓计算换手率 = sum(|new_weight - old_weight|)
      - 交易成本 = 换手率 × 双边成本 0.0026
      - 冲击成本 = 持仓市值 / 日均成交额 × 0.1（线性模型）
      - 净收益 = 毛收益 - 交易成本 - 冲击成本
    outputs:
      - "毛收益 vs 净收益对照表"
      - "年度换手率与成本占比"
      - "成本后净值曲线"
    pass_criteria:
      - "成本后年化净收益 > 5%"
      - "成本后 Sharpe > 0.3"
      - "成本占比 < 30%（成本不吞噬 alpha）"

  module_2_capacity:
    description: |
      容量估算：5000 万资金下的冲击成本。
    method: |
      - 对每个调仓日，计算持仓股的日均成交额
      - 冲击成本 = 持仓市值 / 日均成交额 × 0.1
      - 分档统计：<5% / 5-20% / >20%
      - 5000 万资金下单只持仓约 100 万（50 只），冲击成本应 <5%
    outputs:
      - "持仓市值/日均成交额分档统计"
      - "5000 万资金下冲击成本时序"
      - "容量上限估算（冲击成本 <20% 对应的最大资金）"
    pass_criteria:
      - "5000 万资金下冲击成本 < 净收益 20%"
      - "持仓市值/日均成交额 >20% 的股票数 < 5"

  module_3_crowding:
    description: |
      因子拥挤度监测：F2/F5 因子是否已拥挤。
    method: |
      - 全市场 F2_ep Z-score 均值时序（衡量因子拥挤度）
      - 全市场 F5_vol Z-score 均值时序
      - 纳入股 vs 剔除股的不对称行为
      - 拥挤度分位：当前值在历史分布中的位置
    outputs:
      - "F2/F5 因子拥挤度时序"
      - "当前拥挤度历史分位"
      - "纳入 vs 剔除股行为对照"
    pass_criteria:
      - "F2/F5 拥挤度 < 历史 90 分位"

  module_4_turnover_alpha:
    description: |
      换手-alpha 闭环：边际信息收益 vs 边际交易成本。
    method: |
      - 计算每次调仓的信息收益（新组合 vs 旧组合的下期收益差）
      - 计算每次调仓的交易成本
      - 边际信息收益 = 信息收益 / 换手率
      - 边际交易成本 = 交易成本 / 换手率
      - 闭环：边际信息收益 ≥ 边际交易成本
    outputs:
      - "每次调仓的信息收益与交易成本对照"
      - "边际信息收益 vs 边际交易成本时序"
      - "换手率优化建议"
    pass_criteria:
      - "边际信息收益 ≥ 边际交易成本（换手合理）"

  module_5_position_management:
    description: |
      【核心创新】仓位管理改进：从目标波动率改为最大回撤约束 + 动态止损。
      直接控回撤，不控波动。
    method: |
      仓位规则：
        (1) 最大回撤约束：
            - 跟踪组合从高点的回撤
            - 回撤 > 10% 时：仓位降至 70%
            - 回撤 > 15% 时：仓位降至 50%
            - 回撤 > 20% 时：仓位降至 30%
            - 回撤修复（从低点反弹 5%）时：仓位恢复到 100%
        (2) 动态止损：
            - 组合从高点回撤 25% 时触发止损，仓位降至 20%
            - 止损后需 3 个月观察期才恢复加仓
        (3) F5 信号叠加（修复 Phase 3 bug）：
            - F5 信号 z-score 阈值从 -1 调到 -0.5
            - vol_z < -0.5 时额外减仓 20%
      仓位调节频率：日频（与回测一致）
    variants:
      - {id: V1, name: "纯最大回撤约束（无 F5 叠加）"}
      - {id: V2, name: "最大回撤约束 + F5 信号叠加"}
      - {id: V3, name: "保守版（回撤阈值 8%/12%/18%）"}
    outputs:
      - "V1/V2/V3 vs Baseline 收益风险对照（日频口径）"
      - "仓位时序曲线"
      - "回撤降低幅度对照"
    pass_criteria:
      - "V2 回撤 < 30%（核心目标）"
      - "V2 年化下降 < 5pp（收益-回撤 trade-off 合理）"
      - "V2 Sharpe 不下降"
```
