## 一、执行摘要

Phase 4 V2 五模块全流程跑通。V2 修复方向有效但改善幅度不足以达到 Gate 5 标准。核心结论：

1. **M2 流动性过滤方向有效但未达标**（❌）
   - 高冲击股票数 V1=31 → V2=23（-26%），但仍 > 5（阈值）
   - 流动性过滤是行业标准做法，方向正确，但 1000 万阈值不足以彻底解决
2. **M5 仓位管理接近通过但未达 30% 阈值**（❌）
   - V2 回撤 36.85% → 35.04%（-4.9%），V4 回撤 34.83% < 35% 硬阈值
   - V4 Sharpe 1.0691 > V3 1.0519 > V2 0.9995 > V1 0.8934（方向有效）
   - 仓位均值已降到 0.540，继续收紧会牺牲年化
3. **alpha 三模块稳定通过**（M1/M3/M4 ✅）
   - M1: 年化 15.61%，Sharpe 0.6686，成本占比 1.41%
   - M3: F2=10.81%，F5=78.38%（F5 接近 90% 警戒线）
   - M4: 边际信息收益 0.00579 vs 边际成本 0.00134（4.3x）
4. **仓位管理降波动有效（23%→15%，-37%），降回撤有限（40%→35%，-13%）**
   - 极端回撤来自系统性下跌，仓位管理无法完全消除
   - 这是策略的结构性局限，不是 bug

**终判**：3/5 模块通过，按 spec "≤3 模块通过 → Phase 4 失败，停止迭代" → **V2 失败，停止 V3 迭代**。

## 五、关键发现与解读

### 发现 1：V2 修复方向有效但幅度不足

V2 三处修复（流动性过滤 + F5 阈值 + V4 变体）方向都正确：
- M2 流动性过滤：31 → 23 只（-26%）
- M5 F5 阈值 -0.3：V2 与 V1 出现差异（V2 Sharpe 0.9995 > V1 0.8934）
- M5 V4 激进版：回撤 34.83% < 35% 硬阈值

但改善幅度不足以达到 Gate 5 标准（M2 阈值 5，M5 阈值 30%）。

### 发现 2：M2 是结构性瓶颈

23 只 >20% 持仓，继续提高流动性阈值（1000万→2000万）可能让 M2 通过，但：
- 选股池缩小会衰减 alpha（spec line 222 警告）
- 容量上限 2.32 亿（V1=V2），说明问题是分布不均而非总量不足
- 这是 A 股小盘股流动性陷阱的结构性问题，不是参数调优能解决

### 发现 3：M5 接近通过但 30% 阈值是硬约束

V4 回撤 34.83% 距 30% 阈值差 4.83pp：
- V4 仓位均值 0.540（已激进）
- 仓位管理降波动 -37%，降回撤仅 -13%
- 极端回撤来自系统性下跌（2015 股灾/2018 贸易战/2024 小盘股危机）
- 仓位管理能减仓但躲不过系统性事件，这是策略的结构性局限

### 发现 4：alpha 三模块稳定有效

M1/M3/M4 在 V1→V2 迭代中保持通过：
- M1: 成本占比 1.36% → 1.41%（流动性过滤略增成本，仍极稳健）
- M3: 拥挤度分位完全不变（流动性过滤不影响因子拥挤度）
- M4: 信息收益 4.7x → 4.3x 成本（仍远超阈值）
- **策略 alpha 有效性得到 V1+V2 双重验证**

### 发现 5：V1≈V2 现象在 V2 中消失

- Phase 3 V1=V2 是实现 bug（F5 overlay 阈值 -1 从未触发）
- Phase 4 V1 V1≈V2 是参数过严（F5 overlay 阈值 -0.5 触发太少）
- **Phase 4 V2 V2 ≠ V1**（V2 Sharpe 0.9995 > V1 0.8934，F5 overlay 阈值 -0.3 实际触发）
- 三阶段递进：实现 bug → 参数过严 → 参数合理，F5 overlay 价值得到验证

### 发现 6：仓位管理 V1 < V2 < V3 < V4 单调改进

| 变体 | 阈值 | Sharpe | 回撤 | 仓位均值 |
|------|------|--------|------|---------|
| V1 纯 DD | 10%/15%/20% | 0.8934 | 0.3686 | 0.733 |
| V2 DD+F5(-0.3) | 10%/15%/20% | 0.9995 | 0.3504 | 0.604 |
| V3 保守 | 8%/12%/18% | 1.0519 | 0.3504 | 0.569 |
| V4 激进 | 7%/10%/15% | 1.0691 | 0.3483 | 0.540 |

- Sharpe 单调递增，回撤单调递减
- 仓位管理方向明确有效，但受系统性下跌硬约束

## 六、决策

### 6.1 Phase 4 V2 终判

**3/5 模块通过**，按 spec 失败条件：
- spec line 201 "≤3 模块通过 → Phase 4 失败，停止迭代" → ⚠️ 触发
- spec line 222 "若 M2 仍未通过 → V2 失败，停止迭代" → ⚠️ 触发
- spec line 223 "若 M5 V4 回撤仍 > 35% → 回撤控制无效" → ✅ 未触发（V4 回撤 34.83% < 35%）

**V2 终判**：Phase 4 V2 失败，**停止 V3 迭代**。

### 6.2 停止迭代的理由

1. **严格遵守预注册**：spec 失败条件明确"≤3/5 停止迭代"，不在结果后调整标准（防 data-mining）
2. **M2 是结构性瓶颈**：A 股小盘股流动性陷阱，参数调优无法彻底解决
3. **M5 V4 已是激进极限**：仓位均值 0.540，继续收紧会牺牲年化
4. **V3 迭代预期收益有限**：流动性阈值 1000万→2000万可能让 M2 通过，但 alpha 衰减风险高（spec line 222 警告），且 M5 30% 阈值仍难破
5. **alpha 已得到双重验证**：V1+V2 三模块稳定通过，策略 alpha 有效性不是问题

### 6.3 项目结论

**策略 F2+F5 等权 z-score（P1-F2F5-EP-LowVol-2026Q2-v1）**：
- ✅ **alpha 有效**：FM t=2.95 + DSR 0.97 + 信息收益 4.3-4.7x 成本（Phase 1-3 + Phase 4 V1/V2 五重验证）
- ✅ **对成本极度稳健**：成本占比 1.36-1.41%，年化损失仅 0.74pp
- ✅ **因子未拥挤**：F2 拥挤度 10.81%，F5 拥挤度 78.38%（接近警戒线但未突破）
- ⚠️ **容量受限**：5000 万资金下 23 只高冲击持仓，容量上限 2.32 亿
- ⚠️ **回撤控制接近通过**：V4 回撤 34.83% < 35% 硬阈值，但 > 30% Gate 5 标准
- ⚠️ **极端回撤是结构性局限**：仓位管理降波动 -37%，降回撤仅 -13%（系统性下跌难避）

### 6.4 实盘建议（非 Gate 5 通过路径，人工风控补位）

虽然 V2 未达 Gate 5 通过标准，但策略 alpha 有效 + 仓位管理方向有效，可考虑**小资金 + 人工风控**实盘：

1. **资金规模**：< 2000 万（避开 M2 容量瓶颈，5000 万资金下 23 只高冲击 → 2000 万资金下高冲击数预计 < 10）
2. **仓位管理**：采用 V4 激进版（7%/10%/15% 阈值，仓位均值 0.540）
3. **人工风控补位**：
   - 系统性下跌预警：人工判断是否减仓（仓位管理无法处理的极端事件）
   - F5 拥挤度监控：突破 90% 分位暂停加仓
   - 流动性监控：单只持仓 >20% 日均成交额时人工减仓
4. **预期表现**（基于 V2 V4 数据）：
   - 年化 ~15.64%，Sharpe ~1.07，回撤 ~34.83%
   - 实盘可能因人工风控优于 V4 数据，但需接受回撤可能 > 30%

**重要提示**：此实盘建议偏离 spec 预注册标准（≤3/5 失败），属于人工风控补位路径，需明确告知决策者风险。
