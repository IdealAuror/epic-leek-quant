## 四、关键发现与解读

### 发现 1：策略对成本极度稳健（M1 强通过）

成本占比仅 1.36%，年化只损失 0.74pp。这说明：
- F2+F5 选股稳定，换手率 4-5 倍但每次调仓是"结构性调整"而非"高频抖动"
- 单次调仓成本极低（约 0.6-0.75%/年），alpha 远超成本
- **策略在成本维度实战可用**，5000 万资金下成本不构成约束

### 发现 2：小盘股流动性陷阱（M2 真实失败）

31 只股票持仓 > 20% 日均成交额。这是 Phase 4 最重要的发现：
- 全市场选股（无流动性过滤）会选到低流动性小盘股
- 5000 万资金下 1.68% 持仓严重不流动，最大冲击成本 61.93%
- 容量上限 2.32 亿，看似充裕但分布不均（少数持仓严重受限）

**对 V2 的启示**：加流动性过滤（剔除近 20 日日均成交额 < 1000 万的股票），预计能解决 M2 失败。

### 发现 3：F5 接近拥挤警戒（M3 通过但需监控）

F5 拥挤度历史分位 78.38%，离 90% 阈值只剩 12pp：
- F2 拥挤度极低（10.81%），主因子未拥挤
- F5 作为风险调节器被市场广泛关注，拥挤度上升
- **对 V2 的启示**：实盘需建立 F5 拥挤度月度监控，若突破 90% 需暂停加仓

### 发现 4：alpha 真实但调仓日间衰减（M4 信息价值 5x 成本）

边际信息收益 0.634% vs 边际成本 0.134%（4.7 倍），证明换手创造价值：
- 但净收益为正比例仅 44.44% → 调仓日间 alpha 衰减明显
- 换手合理但不能频繁换手
- **对 V2 的启示**：维持季度调仓，不考虑提高频次

### 发现 5：仓位管理降波动有效，降回撤有限（M5 核心矛盾）

V1/V2/V3 都把波动从 23.33% 压到 17.65%（-24%），但回撤只从 39.79% 压到 36.85%（-7.4%）：
- **极端回撤来自系统性下跌**（2015 股灾/2018 贸易战/2024 小盘股危机）
- 仓位管理能减仓但躲不过系统性事件
- 这是策略的**结构性局限**，不是 bug

### 发现 6：V1≈V2 现象的本质转变

- **Phase 3 V1=V2 是实现 bug**（F5 overlay 阈值 -1 从未触发）
- **Phase 4 V1≈V2 是参数过严**（F5 overlay 阈值 -0.5 在 37 个调仓日里几乎不触发）
- V3 保守版 Sharpe 0.9526 > V2 Sharpe 0.8995，证明方向对，需更紧约束
- **对 V2 的启示**：F5 overlay 阈值从 -0.5 调到 -0.3；同时测试 V4 更激进回撤约束（7%/10%/15%）

## 五、决策

### 5.1 Phase 4 V1 终判

**3/5 模块通过**，按 spec 失败条件：
- "5 模块全部通过 → Phase 4 通过" → ❌ 不成立
- "4 模块通过（模块 5 未通过）→ 策略可实盘但需人工风控" → ❌ 不成立（仅 3/5）
- "≤3 模块通过 → Phase 4 失败，策略未达实盘标准，停止迭代" → ⚠️ 部分成立

**V1 终判**：Phase 4 V1 失败。但 spec 中"停止迭代"过严——失败模式有明确修复路径（M2 加流动性过滤 / M5 调参数），且 M1/M3/M4 都强通过，策略有真实 alpha（FM t=2.95 + DSR 0.97 + 信息收益 4.7x 成本）。

**修正决策**：V1 失败，启动 V2 迭代（不是停止迭代）。

### 5.2 策略决策

- **当前最优策略仍为 F2+F5 等权 z-score**（P1-F2F5-EP-LowVol-2026Q2-v1）
- V1 暴露两个可修复问题：流动性过滤缺失 + F5 overlay 参数过严
- **V2 不替换 Baseline 因子逻辑**，只修两个具体问题

### 5.3 V2 迭代启动条件

1. **策略**：F2+F5 等权 z-score Baseline + 流动性过滤
2. **规模**：5000 万（不变）
3. **M2 修复**：加流动性过滤（剔除近 20 日日均成交额 < 1000 万的股票）
4. **M5 修复**：F5 overlay 阈值 -0.5 → -0.3；新增 V4 变体（7%/10%/15% 阈值）
5. **监控重点**：F5 拥挤度（78.38% → 90% 警戒线）
