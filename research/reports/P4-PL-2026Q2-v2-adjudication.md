## 五、关键洞察

### 洞察 1：V2 修复方向有效但幅度不足

V2 三处修复方向全部正确：
- 流动性过滤：M2 高冲击数 -26%（31→23）
- F5 阈值 -0.3：V2 vs V1 出现差异（Sharpe +0.106）
- V4 激进版：回撤 34.83% < 35% 硬阈值

但 Gate 5 阈值（M2 <5，M5 <30%）是预注册锁定的硬约束，V2 改善幅度不足以达到。

### 洞察 2：M2 是结构性瓶颈

A 股小盘股流动性陷阱是结构性问题：
- 容量上限 2.32 亿（V1=V2），问题是分布不均而非总量不足
- 继续提高流动性阈值（1000万→2000万）会缩小选股池、衰减 alpha
- **参数调优无法彻底解决，需要结构性改变（如分资金规模分级选股）**

### 洞察 3：M5 接近通过但 30% 阈值是硬约束

V4 回撤 34.83% 距 30% 阈值差 4.83pp：
- 仓位均值 0.540 已是激进极限
- 仓位管理降波动 -37%，降回撤仅 -13%
- **极端回撤来自系统性下跌**（2015 股灾/2018 贸易战/2024 小盘股危机）
- 仓位管理能减仓但躲不过系统性事件，这是策略的结构性局限

### 洞察 4：alpha 三模块稳定有效（V1+V2 双重验证）

M1/M3/M4 在 V1→V2 迭代中保持通过：
- M1: 成本占比 1.36% → 1.41%（仍极稳健）
- M3: 拥挤度分位完全不变
- M4: 信息收益 4.7x → 4.3x 成本（仍远超阈值）
- **策略 alpha 有效性得到 V1+V2 双重验证，不是偶然**

### 洞察 5：V1≈V2 现象在 V2 中消失

- Phase 3 V1=V2：实现 bug（F5 overlay 阈值 -1 从未触发）
- Phase 4 V1 V1≈V2：参数过严（F5 overlay 阈值 -0.5 触发太少）
- **Phase 4 V2 V2 ≠ V1**：V2 Sharpe 0.9995 > V1 0.8934（F5 overlay -0.3 实际触发）
- 三阶段递进：实现 bug → 参数过严 → 参数合理，**F5 overlay 价值得到验证**

### 洞察 6：仓位管理 V1 < V2 < V3 < V4 单调改进

Sharpe 单调递增（0.89→1.07），回撤单调递减（36.86%→34.83%），仓位单调递减（0.733→0.540）。**仓位管理方向明确有效，但受系统性下跌硬约束**。

### 6.3 实盘建议（非 Gate 5 通过路径，人工风控补位）

虽然 V2 未达 Gate 5 通过标准，但策略 alpha 有效 + 仓位管理方向有效，可考虑**小资金 + 人工风控**实盘：

1. **资金规模**：< 2000 万（避开 M2 容量瓶颈）
2. **仓位管理**：V4 激进版（7%/10%/15% 阈值，仓位均值 0.540）
3. **人工风控补位**：
   - 系统性下跌预警（人工判断减仓）
   - F5 拥挤度监控（突破 90% 暂停加仓）
   - 流动性监控（单只持仓 >20% 日均成交额时人工减仓）
4. **预期表现**：年化 ~15.64%，Sharpe ~1.07，回撤 ~34.83%

**风险提示**：此实盘建议偏离 spec 预注册标准（≤3/5 失败），属于人工风控补位路径，需明确告知决策者风险。
