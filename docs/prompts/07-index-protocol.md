# Prompt 07 — _index.md 写入协议（高级模型 + Flash 共用）

> 本文件规定 `research/_index.md` 的**写入时机、写入字段、并发防竞态**。
> _index.md 是研究看板，所有 spec/报告/决策/迭代轮次都必须在此登记——这是
> "全量上报防 HARKing"和"分析前先读历史"两条纪律的物理载体。若无本协议，
> 多代理并行时会产生写入竞态，或漏登失败结果导致择优上报。

---

## 一、_index.md 的结构（6 节）

| 节 | 内容 | 写入者 |
|----|------|--------|
| §一 命名规范 | 静态参考，Phase 0 写定，不随迭代变更 | 高级模型（Phase 0） |

| §六 变更日志 | 每次 _index.md 自身变更记录 | 高级模型 |

## 二、写入时机（谁在何时写什么）

### 2.1 spec 落盘时（高级模型）

当高级模型把 spec 标 `locked` 落盘到 `research/specs/<spec_id>.md` 后，**立即**更新 _index.md：

**§二 因子看板**——更新该因子行的 spec_id 与状态：
```
| F1 EV<0 | P1-F1-EV-2026Q2-v1 | 🟡进行中 | 0 | — | 待执行 |
```

**§四 迭代轮次与放弃记录**——新增一行：
```
| P1-F1-EV-2026Q2-v1 | 1 | — | 🟡执行中 | — |
```

### 2.2 Flash 交付时（Flash 提议，高级模型确认）

Flash 交付代码+结果+偏差记录后，**不直接写 _index.md**，而是提议更新内容给高级模型。高级模型审核代码后（03 通过后）确认写入：

**§四 迭代轮次与放弃记录**——更新该 spec_id 行的 IC_IR 与判定：
```
| P1-F1-EV-2026Q2-v1 | 1 | 0.34 | ✅通过 | — |
```

> **为什么 Flash 不直接写**：Flash 不解释结果、不判定通过与否（02 禁线）。
> _index.md 的判定字段是高级模型的判定结论，必须由高级模型写入。

### 2.3 04 判定完成时（高级模型）

04 判定落盘后，**立即**更新 _index.md：

**§二 因子看板**——更新该因子行的状态与最近结论：
```
| F1 EV<0 | P1-F1-EV-2026Q2-v1 | ✅通过 | 1 | IC_IR=0.34, Q1>基准+3% | 进 Phase 2 候选 |
```

或：
```
| F4 国企 | P1-F4-SOE-2026Q2-v2 | ❌放弃 | 2 | 连续两轮 IC_IR<0.2, 控制 size 后失效 | size 代理, 永久放弃 |
```

**§三 待验证假设登记**——更新对应 H# 的结果：
```
| H11 | 国企因子控制 size + low-vol 后仍显著 | §3.3.2 | P1-F4-SOE-2026Q2-v2 | ❌已证伪 |
```

### 2.4 迭代决策时（高级模型）

06 迭代决策落盘后，更新 _index.md：

**§四 迭代轮次与放弃记录**——若放弃，新增放弃原因行：
```
| P1-F4-SOE-2026Q2-v2 | 2 | 0.15 | ❌放弃 | 连续两轮<0.2, size 代理, 永久放弃 |
```

**§五 澄清与迭代落盘**——记录 spec 修订原因：
```
| 2026-06-28 | P1-F1-EV-2026Q2-v1 | spec修订 | 因 cash_equivalents 含受限资金, 新增流动比率>1.5 过滤 → v2 |
```

**§六 变更日志**——记录 _index.md 自身变更：
```
| 2026-06-28 | F1 判定通过, H6 标已证实 | Phase 1 F1 完成 |
```

## 三、并发防竞态（并行模式必备）

当 spec 标注 `parallel.enabled=true` 时，多个子代理并行执行不同切片，可能同时尝试更新 _index.md。

### 规则

1. **子代理（Flash）绝不直接写 _index.md**——见 §2.2，所有 _index.md 写入由高级模型串行执行。
2. **高级模型按 spec_id 串行写入**——即使切片并行跑，高级模型收到各切片结果后**逐个 spec_id 串行更新** _index.md，不批量并发写。
3. **每个切片一个 spec_id**——切片不共享 spec_id，各切片结果在 _index.md §四 各占一行：
   ```
   | P1-F1-EV-2026Q2-v1-CSI300 | 1 | 0.34 | ✅ | 沪深300样本 |
   | P1-F1-EV-2026Q2-v1-CSI500 | 1 | 0.28 | ⚠️ | 中证500样本, IC_IR<0.3 |
   | P1-F1-EV-2026Q2-v1-AllA   | 1 | 0.31 | ✅ | 全A样本 |
   ```
4. **全量上报**——含失败的切片全部登记，**禁止只报通过的切片**。审核时若发现 _index.md 缺失某切片行，视为择优上报嫌疑，退回补齐。

### 写入顺序

```
子代理完成切片 → 各自交付结果+偏差记录 → Flash 归并 → 交付高级模型
    ↓
高级模型串行审核（03）→ 串行判定（04）→ 串行更新 _index.md
```

## 四、字段规范

### §二 因子看板字段

| 字段 | 取值 | 示例 |
|------|------|------|
| 因子 | F1–F5 / MF | F1 EV<0 |
| spec_id（最新） | 见 01 命名规范 | P1-F1-EV-2026Q2-v1 |
| 状态 | ⏳待启动 / 🟡进行中 / ✅通过 / ❌放弃 / ⚠️不确定 | ✅通过 |
| 迭代轮次 | 整数（含变体） | 2 |
| 最近结论 | 一句话 | IC_IR=0.34, Q1>基准+3% |
| 下一步 | 进 Phase 2 / 放弃 / 待变体 | 进 Phase 2 候选 |

### §三 待验证假设字段

| 字段 | 取值 | 示例 |
|------|------|------|
| # | H1–H15 | H6 |
| 假设 | 一句话 | EV<0 因子在 A 股具有显著正向选股能力 |
| 来源 | 文档章节 | §3.1 |
| 验证 spec | spec_id | P1-F1-EV-2026Q2-v1 |
| 结果 | 待验证 / ✅已证实 / ❌已证伪 / ⚠️部分成立 | ✅已证实 |

### §四 迭代轮次字段

| 字段 | 取值 | 示例 |
|------|------|------|
| spec_id | 含切片后缀（若有） | P1-F1-EV-2026Q2-v1-CSI300 |
| 轮次 | 整数 | 1 |
| IC_IR | 数值（NW 标准误） | 0.34 |
| 判定 | 🟡执行中 / ✅通过 / ❌假设错误 / ❌实现错误 / ❌放弃 / ⚠️不确定 | ✅通过 |
| 放弃/继续理由 | 文字 | — |

## 五、防 data-mining 硬规则

1. **全量上报**：含失败切片、含 IC_IR<0.2 的轮次，全部登记，禁止择优。
2. **串行写入**：并行模式下 _index.md 由高级模型串行写，子代理不写。
3. **即时更新**：spec 落盘/交付/判定/决策四个时点立即更新，不积压。
4. **分析前先读**：每轮分析前先读 _index.md 历史结论（agent-workflow 第六机制）。
5. **缺失即择优嫌疑**：审核时发现 _index.md 缺失某切片/某轮次 → 退回补齐。
