# Prompt 06 — 迭代触发器（高级模型·跨因子决策）

> 本文件是 Phase 1 中**因子之间**的决策点提示词。当一个因子的 04 结果判定完成后，
> 高级模型按本文件决定下一步动作。它也是防止"无限迭代到偶然通过"和"重复尝试
> 已放弃方向"的关键纪律。

---

## 一、决策树

一个因子的 04 判定完成后，按以下决策树选择下一步：

```
判定结果 = 通过?
├── 是 → 该因子进 Phase 2 候选清单 → 进下一个因子（05 §二步骤 1）
├── 否（假设错误，真负） → 记录放弃原因 → 进下一个因子 或 进入"变体尝试"
├── 否（实现错误，假负） → 修复实现 → 重跑（不计入"连续两轮"）
└── 不确定 → 明确不确定性来源 → 生成新 spec（新假设）→ 重跑
```

### 三态判定与动作对照

| 04 判定 | 含义 | 下一步动作 |
|---------|------|----------|
| ✅ 通过 | Gate 1 七条全满足 | 进 Phase 2 候选 → 下一因子 |
| ❌ 假设错误（真负） | 因子在 A 股无定价能力 | 记录放弃 → 检查是否触发"连续两轮" → 下一因子 |
| ❌ 实现错误（假负） | 因子可能有效，实现有缺陷 | 修复实现 → 重跑（**不计入连续两轮**） |
| ⚠️ 不确定 | 统计不显著或效应量太小 | 明确不确定性来源 → 新 spec → 重跑 |

---

## 二、"连续两轮"放弃规则（硬性）

PROJECT-PLAN.md §四第 4 条 + agent-workflow 抑制确认偏误第四机制：

> 同一个假设连续两轮 IC_IR < 0.2 → 放弃该因子方向。

### 计数规则

| 情况 | 是否计入"连续两轮" | 说明 |
|------|------------------|------|
| 判定为"假设错误（真负）"且 IC_IR<0.2 | ✅ 计入 | 真负算一轮 |
| 判定为"实现错误（假负）" | ❌ 不计入 | 假负是工具问题，不是假设问题 |
| 判定为"不确定"且 IC_IR<0.2 | ⚠️ 视情况 | 若是样本不足 → 不计入；若是因子衰减 → 计入 |
| 判定为"通过" | 计数清零 | 通过即重置 |

### 放弃后的重启条件

放弃后若要重启该因子方向，**必须**在 `research/_index.md` §四记录：
> "为什么继续迭代这个方向仍合理"——附数据/文献/新假设支撑。

无此记录则永久放弃，Phase 1 不再重试。

## 三、变体尝试规则

当主因子（BASE 版）未通过但仍有合理性时，可尝试 theory-framework §5.2 的变体：

| 因子 | 可尝试变体 |
|------|----------|
| F1 EV<0 | 壳价值剔除版 / 净现金精细化版（挖附注） |
| F2 PE<10 | EP vs BM / 行业分档 PE / FCF/Y 版 |
| F3 分红 | 裸股息率 vs 利润稳定过滤版 / 综合股东回报版 |
| F4 国企 | Fama-MacBeth 控制 size+low-vol 版 / 2019.06 分段版 |
| F5 质量 | ROE 版 / 低杠杆+正OCF 版 / 条件 VaR 版 |

### 变体尝试的限制

1. **变体不是"换参数碰运气"**——每个变体必须有明确的新假设（写在 spec 的 `hypothesis` 字段）。
2. **变体的 IC_IR<0.2 同样计入"连续两轮"**——主因子 + 变体共享计数器。
3. **主因子通过后不再跑变体**——变体是 Plan B，不是锦上添花。
4. **变体结果全量上报**——含失败的变体全部进 `_index.md`，禁止择优。

## 四、因子间衔接决策

完成一个因子后，进下一个因子前，检查以下依赖：

### 相关性检查（进入 Phase 2 前必做）

每完成一个因子，在 `_index.md` 记录它与已通过因子的 Spearman Rank Correlation：

| 因子对 | Spearman Corr | 判定 |
|--------|--------------|------|
| F1 ∩ F2 | ? | > 0.6 → 重复暴露，Phase 2 需去重 |
| F2 ∩ F3 | ? | > 0.6 → 重复暴露 |
| F4 ∩ F5 | ? | > 0.6 → 重复暴露 |

### 因子顺序的动态调整

正常按 F1→F2→F3→F4→F5 推进，但以下情况可调整：

| 情况 | 调整 |
|------|------|
| F1 因数据层问题卡住（如 cash_equivalents 字段不准） | 暂停 F1，先跑 F2 验证数据层 |
| F1 严重壳价值污染，需补"壳价值净化版" | F1 标记"待变体"，继续 F2，变体结果回来再判 F1 |
| F4 的 Fama-MacBeth 需要 F5 的 low-vol 数据 | 先跑 F5 的 low-vol 代理变量，再跑 F4 |

### Phase 1 提前结束条件

以下情况 Phase 1 可提前结束，进入 Phase 2：

1. **5 个因子全部判定完成**（通过或放弃）——正常结束。
2. **通过的因子 ≥ 3 个**且放弃的因子已记录原因——可带着 ≥3 因子进 Phase 2。
3. **通过的因子 < 2 个**——策略假设整体存疑，**不进 Phase 2**，回到 theory-framework 重新审视五大因子的整体合理性。

## 六、防 data-mining 硬规则

1. **不因结果调整通过标准**——Gate 1 七条锁定，结果出来后不改。
2. **连续两轮硬放弃**——不靠"再试一次说不定就过了"。
3. **变体不择优**——主因子 + 变体全量上报，含失败。
4. **相关性必记录**——为 Phase 2 去重提供依据。
5. **分析前先读历史**——避免重复尝试已放弃方向。
6. **所有决策有数据支持**——禁用"根据经验"。
