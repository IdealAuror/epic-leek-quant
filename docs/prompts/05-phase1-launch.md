> 本文件是 Phase 1（单因子验证）的**总指挥提示词**。高级模型在 Phase 1 启动时
> 加载本文件，按因子顺序逐个推进，每个因子走完整的 spec→执行→审核→判定 闭环
> （见 `01`–`04`），因子之间按 `06-iteration-trigger.md` 决策衔接。
>
> Phase 1 的整体目标：逐一验证五大因子在 A 股的截面选股能力，通过 Gate 1 的
> 因子进入 Phase 2 多因子合成，未通过的记录原因并放弃/降级。

## 一、Phase 1 总览

### 因子验证顺序（PROJECT-PLAN.md §三 Phase 1）

```
F1 EV<0 → F2 盈利收益率(PE) → F3 股东回报(分红+回购) → F4 国企背景 → F5 财务质量
```

**为什么是这个顺序**：
- F1 是最极端、最可能无效（壳价值污染）的因子，先跑能最快暴露数据层与撮合层问题；
- F1 的结论影响 F2（同为价值因子，需测相关性）；
- F3 依赖 F1/F2 的"价值池"做分层（裸股息率 vs 价值池内股息率）；
- F4 是最可能被 F5/size 代理的，放后段便于控制变量；
- F5 作为下行保护因子，需在前面因子的极端回撤区间做条件 VaR。

### Phase 1 整体 Gate 1（PROJECT-PLAN.md §三）

每个因子必须同时满足以下 5+2 条才能标"通过"：

| # | 规则 | 来源 |
|---|------|------|
| 1 | Rank IC_IR > 0.3（NW 标准误，滞后 4 期） | PROJECT-PLAN |
| 2 | Q1 多头组年化收益 > 基准年化 + 2% | PROJECT-PLAN |
| 3 | Q1–Q5 分组单调 | PROJECT-PLAN |
| 4 | 三样本（沪深300/中证500/全A）表现方向一致 | PROJECT-PLAN |
| 5 | 行业中性化 IC 与市值中性化 IC 均显著 | PROJECT-PLAN |
| 6 | **壳价值净化**：EV<0 因子必须报告剔除小市值后的 IC | theory-framework §4.1 |
| 7 | **制度断点一致性**：所有因子必须以 2019.06 为断点报告前后两段 IC 方向 | theory-framework §4.4 |

规则 6–7 是 A 股专属硬性检查（见 `research/theory-framework.md` §五）。

## 二、每个因子的启动流程

对于 F1–F5 中的每一个因子，高级模型按以下步骤推进：

### 步骤 1：读历史结论

- 读 `research/_index.md` 该因子的历史状态、待验证假设（H1–H15）、迭代轮次。
- 若该因子已有"放弃"记录 → 不重启，除非记录了"为什么继续仍合理"。
- 若该因子已有"通过"记录 → 不重跑，直接进 Phase 2。

### 步骤 2：设计 spec

- 按 `01-research-spec-design.md` 模板写 spec，落盘到 `research/specs/<spec_id>.md`。
- spec 必须回应 `research/thinking-prompt.md` 对该因子的**全部深层追问**：
  - F1 EV<0：壳价值污染、受限资金、A+H 跨市场
  - F2 PE<10：银行板块污染、动态阈值、BM vs EP
  - F3 分红+回购：填权概率、分红税、回购无区分度、价值陷阱
  - F4 国企：内生性、size/low-vol 代理、注册制前后
  - F5 质量：下行保护 vs alpha、与 low-vol 重叠、ROE vs ACC/NOA
- spec 必须包含 Gate 1 的全部 7 条预注册通过标准。
- spec 必须标注是否启用并行（跨样本三切片天然可并行）。
- 设计完按 `01` 末尾的自检清单逐项打勾，全部满足才标 `locked`。

### 步骤 3：派发 Flash 执行

- 把 `locked` 的 spec 交给 Flash，按 `02-flash-execution.md` 执行。
- 若 spec 标注 `parallel.enabled=true`，Flash 按切片维度派发多代理。
- 等待 Flash 交付：执行代码 + 结果 + 偏差记录。

### 步骤 4：代码审核（看结果之前）

- 按 `03-code-review-gate.md` 9 大类清单逐项审核。
- **审核时不得查看 `results/<spec_id>/` 下的任何数字**。
- 审核结论落盘到 `research/reports/<spec_id>-code-review.md`。
- P0 问题退回 Flash 修复，循环 03→02→03 直到通过。

### 步骤 5：结果判定

- 代码审核通过后，按 `04-result-adjudication.md` 逐条对照 Gate 1 的 7 条规则判定。
- 判定结论落盘到 `research/reports/<spec_id>-adjudication.md`。
- 判定为"通过"/"不通过"/"不确定"三态之一。

### 步骤 6：迭代触发

- 按 `06-iteration-trigger.md` 决定下一步：进下一个因子 / 回头迭代 / 放弃 / 新假设。
- 无论结果如何，按 `07-index-protocol.md` 更新 `research/_index.md`。

## 五、防 data-mining 硬规则（贯穿 Phase 1）

1. **预注册不可调**：Gate 1 的 7 条规则在 spec `locked` 时锁定，结果出来后不调整。
2. **代码审核先于结果分析**：看任何数字前完成 03 门禁。
3. **连续两轮 IC_IR<0.2 → 放弃**：该因子方向永久放弃，除非记录"为什么继续仍合理"。
4. **全量上报**：并行切片结果（含失败）全量进 `_index.md`，禁止择优。
5. **DSR 兜底**：Phase 1 参数扫描必须计算并报告 DSR。
6. **分析前先读历史**：每轮分析前先读 `_index.md` 历史结论，避免重复尝试。
7. **所有结论有数据支持**：禁用"根据经验"作判断依据。
